If you want to try a more advanced technique keep reading to learn about SQL Injection.

SQL Injection is a vulnerability when the user input can be directly used to ask or query the database for information. As an example, when you login to a website using a username and password, this is what the backend looks like:

SELECT * FROM [table] WHERE username=username AND password=password;

This is called SQL code and it checks a list of database records for any record with the username and password you gave, that’s how it finds your account. If a programmer doesn't write their code securely though we can take advantage of this process and perform an attack called SQL Injection.

Instead of typing in a username and password, we can type in code directly into the username field to trick the application. One example of this is:

' OR '1'='1 --

If you type that exactly into the username field, an application reads that as:

SELECT * FROM [table] WHERE username = '' OR '1' = '1' -- AND password =

Translating this gives us: Return all rows in the database where username = whatever OR 1=1. Since 1 always equals 1, it returns a random record in the database, giving us access to the application.

The minus signs are a comment in SQL and essentially tell the code to ignore everything after that point.
